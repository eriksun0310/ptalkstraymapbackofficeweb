# Claude Code 專案指引

## 語言設定

- 全部使用**繁體中文**回覆

## 溝通模式

**核心理念**：當使用者說「**分析給我**」或「**分析和我討論**」時，表示只需要提供分析和建議，不要直接修改程式碼。

**執行準則**：
- 先分析問題的根本原因
- 提供多個解決方案並比較優缺點
- 給出明確的建議和理由
- 等待使用者確認後再進行實作

## 程式碼品質要求

### 禁止行為

- **絕對不要**為了「快速解決」而使用低效方案
- **絕對不要**用補丁方式修復架構問題

### 避免重複程式碼 (DRY 極致執行)

```
發現重複 → 立即重構
寫新功能 → 先檢查現有程式碼
三次法則 → 出現三次必須抽取
```

### 其他原則

- 優先考慮可維護性和可讀性
- 遵循專案既有的程式碼風格
- 不過度工程化，保持簡潔

## 設計風格

- 極簡風格：黑、白、灰 + 單一主色（藍 `#4A90E2`）
- 策略性使用主色標記重點（標題、數字、互動元素）
- 避免過多顏色造成視覺混亂
